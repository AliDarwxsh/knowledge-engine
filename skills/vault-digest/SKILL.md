# vault-digest — Weekly Synthesis

> The weekly intelligence report. Reads what you thought about for 7 days and tells you what none of those notes could say alone.

---

## When to Invoke

- User: "/digest", "/vault-digest", "Weekly summary", "What changed this week?"
- Auto: weekly-digest cron (Sunday 6:00pm)

---

## Execution — Five Phases

### Phase 1: Gather The Week

1. Find all notes with `created:` date within last 7 days (or last sync, whichever is more recent)
2. Find all notes modified in last 7 days
3. Read full content of each — no summaries, full text
4. Build a working set: typically 10-50 notes

### Phase 2: Identify Themes

Cluster notes by:

1. **Recurring concepts** (terms appearing in 3+ notes)
2. **Domain clusters** (notes tagged with same domain)
3. **Question patterns** (similar questions asked across notes)
4. **Decision trajectories** (related decisions over time)

Output: "This week's themes:
- **<Theme 1>** — <N> notes, key question: <question>
- **<Theme 2>** — <N> notes, key question: <question>
- **<Theme 3>** — <N> notes, key question: <question>"

### Phase 3: Generate Cross-Domain Synthesis

This is the core value: find patterns that span multiple domains.

For each pair of themes, ask:
- Where do they intersect?
- What does the connection reveal?
- Is there a higher-order insight that neither theme contains alone?

Example output structure:

```
## Cross-Domain Insight: <Title>

**From**: [[note-1]] (domain: <X>) + [[note-2]] (domain: <Y>)

The intersection: <synthesis statement>

**Implication**: <what this means for the user's work/thinking>

**Evidence**: 3 notes support this, 0 contradict.
```

Generate 3-7 cross-domain insights per digest. Quality > quantity.

### Phase 4: Surface Contradictions & Tensions

Same detection as [[vault-sync]] Phase 4, but with the week's specific context:

1. Are there opposing views held by the same user in different contexts?
2. Did the user's view evolve this week? (older note contradicts newer note)
3. Are there unresolved tensions worth flagging?

Output: "Tensions surfaced this week:
- [[note-a]] vs [[note-b]] on <topic>
- Your view on <X> shifted from <old position> (in [[old-note]]) to <new position> (in [[new-note]])"

### Phase 5: Update Synthesis Hub + Write Digest

1. Append new insights to `wiki/synthesis/00-synthesis-engine-hub.md`
2. Append new tensions to `wiki/journal/contradictions/` (one note per)
3. Write the digest to `wiki/journal/sync-logs/YYYY-MM-DD-digest.md`
4. If permission level allows, also write a digest-summary.md to `11 Timeline/` for quick reference

## Cost Estimation

- Tokens per digest: ~8,000-20,000 (reads full week content)
- Weekly run: ~$0.05-0.15/week with hosted models
- Local models: $0

---

## What This Skill Does NOT Do

- Never modifies individual notes (only appends to synthesis hub and writes the digest itself)
- Never edits `schema/` or `sources/`
- Never auto-resolves contradictions (only surfaces them)
