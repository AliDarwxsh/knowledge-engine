# vault-morning-brief — Daily Context Injection

> Wake up to a brain that organized itself overnight. The first thing you read each morning.

---

## When to Invoke

- Auto: morning-brief cron (7:00am local time)
- User: "/brief", "/vault-brief", "What's new?", "Morning brief"

---

## Execution — Five Phases

### Phase 1: Gather Yesterday's Activity

1. Find all notes with `created:` = yesterday
2. Find all notes with `modified:` = yesterday
3. Find all items processed from `00 Inbox/` yesterday
4. Find all cron job outputs from yesterday (`wiki/journal/sync-logs/`)
5. Count: notes added, notes modified, items processed, contradictions flagged

### Phase 2: Identify Open Loops

Open loops = things started but not finished:

1. Find notes with `status: in-progress` or `status: blocked` in frontmatter
2. Find project notes (`03 Projects/`) not modified in 14+ days (stalled)
3. Find commitments in `06 Decision Archive/` with deadlines approaching (within 7 days)
4. Find contradictions flagged but unresolved in `wiki/journal/contradictions/`
5. Find forward references accumulated but not triaged

### Phase 3: Surface Relevant Context

Patterns that might matter today:

1. **Recurring projects**: Active projects mentioned in recent notes
2. **Unresolved questions**: Questions asked in recent notes without follow-up
3. **Recent principles**: New principles extracted in last 7 days (re-read for relevance)
4. **Pending reviews**: Notes flagged `## Review` section awaiting user confirmation
5. **Daily note check**: Did user complete yesterday's EOD reflection? If not, surface it.

### Phase 4: Flag Attention Items

Three priority levels:

**🔴 Needs attention today**:
- Overdue commitments (deadline passed)
- Contradictions blocking decisions
- Forward references with 10+ incoming references (creating bottleneck)

**🟡 Worth knowing**:
- Stalled projects (14+ days inactive)
- Recurring themes emerging
- New contradictions (not urgent)

**🟢 Just informational**:
- New notes in knowledge library
- Cross-domain insights from digest
- Vault health changes

### Phase 5: Write The Brief

Save to `11 Timeline/YYYY-MM-DD-morning-brief.md` AND deliver as chat message.

Structure:

```markdown
---
title: "Morning Brief: <YYYY-MM-DD>"
summary: "<N> notes yesterday. <M> open loops. <K> need attention."
date: <YYYY-MM-DD>
tags: [brief, daily, timeline]
---

# Morning Brief: <Day, Date>

> <One-line orientation: what kind of day this looks like>

---

## Yesterday

- <N> notes captured
- <M> notes modified  
- <K> inbox items processed
- <vault health score>

## What Moved

- <1-3 sentences on biggest shifts in the vault>

## 🔴 Needs Attention Today

- <item 1>
- <item 2>

## 🟡 Worth Knowing

- <item 1>
- <item 2>

## 🟢 Informational

- <item 1>

## Open Loops

- <project> — last touched <date> — next action: <X>
- <decision> — deadline <date>

## Today's Context

Based on patterns from the last 30 days, today looks like:
- <typical activity pattern for this day of week>
- <recent topics that might be relevant>

## One Question to Carry

<If the digest from last week surfaced open questions, surface the most relevant one>

---

*Generated by vault-morning-brief v1.0.0*
```

## Delivery

This skill has TWO outputs:

1. **File**: `11 Timeline/YYYY-MM-DD-morning-brief.md` (always)
2. **Chat notification**: Brief 4-6 line summary delivered via Hermes notification channel

Chat format:

```
☀️ Morning Brief: <Day, Date>

Yesterday: <N> notes captured, <M> processed.

🔴 <1 most urgent item>
🟡 <1-2 important items>

Open loops: <count>
Vault health: <score>/100

Full brief: [[YYYY-MM-DD-morning-brief]]
```

---

## Cost Estimation

- Tokens per brief: ~2,000-5,000 (reads yesterday's activity)
- Daily run: ~$0.01-0.03/day
- Local models: $0

---

## What This Skill Does NOT Do

- Never modifies other notes
- Never deletes anything
- Never auto-resolves attention items (only surfaces them)
