# 01 Identity

> Identity is a verb. It updates with every decision.

---

## Purpose

This folder holds the operator's **living self-model** — values, narrative arcs, evolving roles, and the principles they refuse to compromise. Identity here is not a static bio. It is a continuously updated document that grows with the operator.

The Human Layer (01 Identity) is the **narrative**, self-reflective view. The Machine Layer ([[wiki/entities/self/identity-moc|Identity MOC]]) is the **structured**, machine-queryable view.

---

## What Lives Here

| Note Type | Purpose | Example File |
|-----------|---------|--------------|
| Identity statement | One-paragraph current self-description | `identity-statement.md` |
| Values | Top-five non-negotiable values | `values-top-five.md` |
| Roles | Current life roles (operator, learner, etc.) | `current-roles.md` |
| Narrative arc | Long-form life story in chapters | `narrative-arc.md` |
| Identity questions | Open questions about who the operator is becoming | `open-questions.md` |

---

## When to Open

- When facing an identity-shaped question (who am I becoming?)
- During a quarterly self-review
- After a major life event or role transition
- When [[02 Principles|02 Principles]] feels misaligned with action
- When [[06 Decision Archive|06 Decision Archive]] reveals contradictions

---

## How to Use

1. **Capture freely.** Drop observations about yourself here without editing.
2. **Reconcile quarterly.** Move stable claims into [[wiki/entities/self/identity-moc|Identity MOC]].
3. **Update in place.** Identity statements get edited, not versioned. The previous state lives in [[11 Timeline|11 Timeline]] under the date of revision.
4. **Cross-link principles.** Every identity claim should link to the [[02 Principles|principles]] it depends on.

> [!warning] **Identity notes are human-only.** AI agents may read but must not write here. See [[schema/agents|agents.md]] for permission boundaries.

## Editing Discipline

- Update identity claims in place.
- Mark each revision with a date in the note footer.
- Move obsolete claims to [[11 Timeline|11 Timeline]] rather than deleting.
- Never overwrite the original — append a new dated section.
